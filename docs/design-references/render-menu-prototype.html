Render menu — design explorations

A — Grid

Two-column thumbnail grid. Each style is a 3:2 mini-poster with a serif name and uppercase one-word descriptor below the photo (same type language as E). Selected state is a cream ring plus a checkmark in the top-right. Categories persist as section headers.

Best for scanning a lot at once
Cost taller scroll than E

B — Editorial list

Single-column horizontal cards — big photo on the left, name plus a one-line description of what the style actually does. Reads like a magazine index. Selected state turns the whole card into a backlit cream chassis.

Best for explaining the style
Cost lots of scroll

C — Filmstrip rails

Each category is a horizontal scroll rail of 3:4 poster-tiles. You scroll sideways inside each section — Apple TV / Spotify shelf feel. Selected tiles get a cream glow plus "Selected" badge.

Best for browsing / category discovery
Cost more interaction per style

E — Wide filmstrip

Like C, but tiles are 3:2 landscape (matching the photos' native ratio) at ~260px wide. Two-and-a-half tiles peek on a 560-wide sheet, three-and-a-half at 720 — strongly implies "scroll me". Reads more cinematic than C, less dense than A.

Best for big, confident preview at a glance
Cost fewer tiles visible per row

D — Large previews

One column, each card is nearly the full sheet width. The thumbnail is big enough to read texture, grain, and palette at a glance. Label plus one-line description below the photo in clean type. More scroll, but every style gets room to make its pitch.

Best for committing to a style
Cost most scrolling of the four
